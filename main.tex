\begin{abstract}
Vehicle-to-Everything (V2X) communications and edge AIoT applications rely on strictly bounded, low-latency processing. However, resource-constrained edge nodes are highly vulnerable to presentation-layer resource exhaustion, where recursive payload structures trigger algorithmic complexity attacks that cause severe Head-of-Line (HoL) blocking. To satisfy real-time safety constraints without inflating continuous inspection overhead, this paper proposes a lightweight, closed-loop digital twin architecture that computationally mirrors the physical V2X data-path and decouples the physical fast-path C++ data plane from a Python cognitive plane. We introduce a Bounded Streaming Structural Risk Filter (B-SSRF), an ultra-lightweight $\mathcal{O}(1)$ anomaly detector leveraging information-theoretic frequency moments (AMS Sketch/$F_2$) to identify malformed payloads prior to expensive deserialization. To optimize this filter dynamically, we model the traffic adaptation problem as a Constrained Markov Decision Process (CMDP). A Deep Q-Network (DQN) agent utilizes a Lagrangian-constrained reward function to modulate a Probabilistic Risk Budget Finite State Machine (PRB-FSM), effectively balancing edge resource utilization with security constraints. Extensive evaluations demonstrate that our mitigation strategy reduces P99 tail latency from 0.3478 ms to 0.0503 ms under high-intensity attacks, achieving microsecond-level edge inference and successfully containing the computational amplification surface.
\end{abstract}

\section{Introduction}

Cooperative Intelligent Transport Systems (C-ITS) rely on Vehicle-to-Everything (V2X) communications for safety-critical, low-latency applications \cite{3GPP_URLLC, Yang_URLLC_2020}. While Multi-access Edge Computing (MEC) offloads complex processing \cite{ETSI_MEC, Mao_MEC_2017}, the presentation-layer parsing of messages (e.g., CAM \cite{ETSI_CAM}, DENM \cite{ETSI_DENM}) via ASN.1 facilities \cite{ETSI_ASN1} and Octet Encoding Rules (OER) \cite{ITU_OER} remains a critical bottleneck. Specifically, under the widely deployed ETSI TS 103 097 V1.2.1 (Security V2) specification \cite{ETSI_SEC_103097_121}, the schema inherently permits unbounded structural nesting of certificate chains. Although ETSI TS 103 097 V1.3.1 / V2.1.1 (Security V3) structurally flattens this schema by constraining the signer identifier to exactly one certificate, rendering it immune by design \cite{ETSI_SEC_103097_131, ETSI_SEC_103097_211}, widely deployed legacy edge nodes remain hard-compiled to V2. By crafting payloads with deeply nested certificates, attackers can trigger exponential object tree expansion. In resource-constrained edge workloads, these recursive "structural bombs" exhaust CPU cycles prior to semantic validation, inducing severe Head-of-Line (HoL) blocking and degrading QoS for legitimate traffic.

Because upgrading legacy V2X firmware is often cost-prohibitive, establishing a protocol-agnostic pre-parsing mitigation layer is essential. However, conventional Deep Packet Inspection (DPI) introduces continuous inspection overhead that inflates tail latency. To resolve this conflict, we propose a lightweight, decoupled closed-loop digital twin architecture (Figure \ref{fig:architecture}) that computationally mirrors the physical vehicular data-path. We combine an $\mathcal{O}(1)$ Streaming Structural Risk Filter (SSRF) in the physical C++ data plane with a Probabilistic Risk Budget FSM (PRB-FSM) optimized by a Deep Reinforcement Learning (DRL) agent in the digital Python cognitive plane. By shifting the security objective from static payload matching to the dynamic mitigation of computational exposure, we adaptively contain adversarial workloads before they reach the expensive ASN.1 parser.

Aligning with the rigorous constraints of AIoT edge computing, the primary contributions of this paper are threefold:
(1) \textbf{Lightweight Algorithmic Anomaly Detection (B-SSRF):} We propose an $\mathcal{O}(1)$ bounded-cost anomaly detector leveraging information-theoretic frequency moments (AMS Sketch/$F_2$). This enables protocol-agnostic identification of structurally anomalous certificate chains without invoking expensive deserialization processes.
(2) \textbf{Two-Tier Digital Twin Architecture:} We design an edge-native framework that decouples a fast-path C++ data plane from a cognitive Python slow path, bridged by a custom sub-microsecond, zero-allocation IPC protocol to enable real-time online training.
(3) \textbf{Lagrangian-Constrained MDP Optimization:} We model the traffic adaptation problem as a Constrained Markov Decision Process (CMDP \cite{Altman1999_CMDP}). Driven by a Deep Q-Network (DQN), our framework dynamically optimizes a probabilistic sampling risk budget via a Lagrangian reward function, effectively balancing edge resource utilization with security requirements.


\section{Background and Threat Model}

\subsection{V2X, ASN.1, and OER Vulnerabilities}
V2X safety applications (e.g., CAM, DENM) rely on Multi-access Edge Computing (MEC) for low-latency processing, presenting complex verification challenges \cite{Sabella_2018}. However, OER's recursive structures—commonly found in IEEE 1609.2 certificates \cite{IEEE_1609_2}—inherently introduce intensive object allocation and CPU parsing overhead at edge nodes, exposing presentation-layer parsers to algorithmic complexity attacks \cite{CWE_674, Wustholz2017, Wang2024}. While Security V3 mitigates this by structurally flattening the ASN.1 schema to eliminate nesting \cite{ETSI_SEC_103097_131, ETSI_SEC_103097_211}, widely deployed legacy roadside and on-board units (RSUs/OBUs) running Security V2 \cite{ETSI_SEC_103097_121} remain fundamentally vulnerable to edge-level denial-of-service \cite{Zhang_Edge_2018}. Because upgrading firmware across operational infrastructure is cost-prohibitive \cite{Farooq2019}, establishing a protocol-agnostic, edge-based mitigation is critical.

\subsection{Threat Model and Amplification}
We assume a computationally bounded adversary injecting syntactically valid, MTU-constrained OER recursion bombs under Security V2. To maximize parsing delay ($L = L_{queue} + C_{parse} + C_{inspection}$), the attacker exploits repetitive structural markers (e.g., \texttt{0x02} octets) to trigger exponential memory allocations. This exhausts CPU cycles \textit{prior} to semantic validation (e.g., MDS \cite{vanderHeijden2023}), inducing Head-of-Line (HoL) blocking and degrading QoS for time-sensitive traffic. Given a strict V2X latency budget ($L \le 10$ ms \cite{3GPP_URLLC}), traditional Deep Packet Inspection (DPI) is computationally prohibitive. 

Our security objective is to bound pre-parsing inspection to $\mathcal{O}(1)$, utilizing the fact that malicious structural repetition sharply contrasts with the high-entropy cryptographic fields of legitimate payloads. Volumetric DDoS, cryptographic compromises, and Sybil attacks are considered out-of-scope.

Recently, Machine Learning (ML) and Deep Reinforcement Learning (DRL) have been adopted for adaptive cyber defense in IoT networks \cite{SACD_2025}. However, conventional ML-based defenses typically rely on payload-heavy feature extraction (e.g., DPI), introducing unacceptable processing latency for V2X safety applications. We bridge this gap using a digital twin architecture that decouples AI decision-making from payload scanning. Rather than using AI for inline packet classification, we employ a DRL agent strictly to modulate the risk budget of a lightweight, $\mathcal{O}(1)$ pre-parsing circuit breaker, achieving robust security without degrading tail latency.

To balance extreme security against CWE-674 structural attacks with the stringent low-latency constraints of V2X communications, we propose a bounded-cost pre-parsing framework. The system provides bounded-cost inspection prior to ASN.1 decoding.

\subsection{Bounded Streaming Structural Risk Filter (B-SSRF)}
We calculate the localized second frequency moment ($F_2$) of byte frequencies within a sliding window to estimate structural instability. This metric, denoted as $SQ$, serves as a collision concentration statistic:
$$SQ = \sum_{j \in \mathcal{V}} c_j^2$$
where $c_j$ is the frequency of byte value $j$ in the window. For legitimate high-entropy V2X payloads, $c_j$ is dispersed, resulting in a low $SQ$. Conversely, OER recursion bombs rely on concentrated structural markers (e.g., choice indices) within MTU constraints, inflating the $F_2$ moment. This is commonly used in streaming algorithms to estimate collision concentration in bounded memory \cite{Alon1996, Krishnamurthy2003}. % TODO: Verify Alon 1996 (AMS sketch) and Krishnamurthy 2003 sketch-based change detection papers
Repeated structural expansion markers in recursive payloads cause the local symbol distribution to concentrate, significantly increasing $SQ$ compared to standard CAM/DENM traffic. While raw sensor data within V2X payloads may occasionally contain local low-entropy fragments (e.g., continuous zero-padding or repeated GPS fields), the structural interleaving of IEEE 1609.2 certificate headers, signatures, and standard OER length determinants prevents the microscale sliding window ($w=64$) from ever sustaining uniform repetition. Under normal conditions, natural structural patterns yield a peak $SQ$ far below the anomaly threshold ($\theta_{\text{sq}} = 600$), rendering false positives extremely unlikely. This structural robustness is empirically validated by the 0.00\% False Positive Rate (FPR) achieved across all test sets.

Algorithm \ref{alg:bssrf} defines this bounded filter. Importantly, the correctness and bounding properties of the framework rely strictly on the mathematical properties of the $F_2$ estimator, avoiding brittle, protocol-specific signatures. By evaluating localized structural variance rather than static patterns, the detector maintains generalizability against evasive MTU-compliant recursion bombs.

\begin{algorithm}[!ht]
\caption{Bounded Streaming Structural Risk Filter (B-SSRF)}
\label{alg:bssrf}
\begin{algorithmic}[1]
\Require Payload $P$, window size $w$, scanning limit $L_{max}$, threshold $\theta$
\State Initialize frequency sketch $H[256] \gets 0$
\State Initialize sliding window $W$
\State $SQ \gets 0$ \Comment{Localized $F_2$ moment}
\State $n_{scan} \gets \min(|P|, L_{max})$

\For{$i = 0$ to $n_{scan} - 1$}
    \State $b \gets P[i]$
    \If{$|W| = w$}
        \State $b_{old} \gets W.\text{pop\_front}()$
        \State $SQ \gets SQ - (2H[b_{old}] - 1)$
        \State $H[b_{old}] \gets H[b_{old}] - 1$
    \EndIf
    \State $W.\text{push\_back}(b)$
    \State $H[b] \gets H[b] + 1$
    \State $SQ \gets SQ + (2H[b] - 1)$ \Comment{Amortized $F_2$ update}

    \If{$|W| = w$ and $SQ > \theta$}
        \State \Return \textbf{True} \Comment{Structural anomaly detected}
    \EndIf
\EndFor
\State \Return \textbf{False}
\end{algorithmic}
\end{algorithm}

This filter bounds the per-packet processing complexity to $\mathcal{O}(w)$ for a constant window size $w$. By restricting the scan to $L_{max}$ bytes, the filter overhead is strictly $\mathcal{O}(1)$ relative to the packet length $N$, consuming negligible CPU cycles compared to standard ASN.1 object allocation.

\subsection{B-SSRF Security Bound and Entropy-Depth Proof}

Although $F_2$ frequency moment estimation is inherently generalized for detecting low-entropy structural repetition, a sophisticated adversary might attempt to evade this filter by injecting high-entropy cryptographic fields (e.g., random ECC public keys or signatures) to dilute the collision concentration within the sliding window. We formally prove that such an evasion attempt is constrained by the physical network capacity, establishing a zero-sum Pareto frontier between evasion capability and exploitation depth.

To explain the source of the entropy budget required for evasion, we first introduce Lemma 1.

\textbf{Lemma 1 (Marker Spacing Requirement).}
\textit{
Consider a sliding inspection window of length $w$.
Suppose an adversary wishes to guarantee that no inspection window simultaneously contains two consecutive recursive structural markers.
Then the spacing between adjacent structural markers must satisfy
}
\begin{equation}
L_{\mathrm{spacing}} \ge w .
\label{eq:lemma1}
\end{equation}

\textbf{Proof.}
Assume the contrary that the spacing between two consecutive structural markers satisfies
$L_{\mathrm{spacing}} < w$.
Since the detector continuously evaluates every sliding window of length $w$, there always exists a window whose left boundary starts at the first marker and whose right boundary extends beyond the second marker.
Consequently, both markers are simultaneously contained within the same inspection window.
Therefore, any adversarial construction with
$L_{\mathrm{spacing}} < w$
cannot guarantee that all sliding windows avoid observing two consecutive markers.
Hence a necessary condition for avoiding simultaneous observation of adjacent structural markers is
\[
L_{\mathrm{spacing}} \ge w .
\]
\hfill $\blacksquare$

\textbf{Corollary 1 (Entropy Budget Requirement).}
\textit{
If the adversary uses random cryptographic bytes as the primary mechanism to separate consecutive structural markers, the required entropy padding satisfies
}
\begin{equation}
S_{\mathrm{entropy}}
\ge
L_{\mathrm{spacing}}
\ge
w.
\label{eq:corollary_entropy}
\end{equation}

The corollary follows directly from Lemma~1. When random padding is used as the mechanism for increasing the spacing, the padding size must be at least the spacing itself.

\textbf{Theorem 1 (Adversarial Capability Bound under MTU Constraints).} \textit{Let $M$ denote the Maximum Transmission Unit (MTU) of the physical network link. Let $S_{\text{syn}}$ represent the minimum structural bytes required by the encoding schema per recursive nesting level. If an adversary injects $S_{\text{entropy}}$ bytes of high-entropy padding per level to evade frequency-based detection under Corollary 1, the maximum achievable recursion depth $D_{\max}$ is strictly bounded by:}
\begin{equation}
D_{\max} \le \frac{M}{S_{\text{syn}} + S_{\text{entropy}}} \le \frac{M}{S_{\text{syn}} + w}
\label{eq:entropy_depth_bound}
\end{equation}

\textbf{Proof.}
To trigger algorithmic complexity exhaustion via structural recursion, each nesting layer must be explicitly encoded within the payload. The minimum payload size required to reach depth $D$ is $D \cdot (S_{\text{syn}} + S_{\text{entropy}})$. For ETSI ITS Security V2, each OER-encoded SecuredMessage is decoded as a single message object and therefore must fit within the effective payload size of one transmitted frame.
Accordingly, the available payload budget is bounded by the
link-layer MTU limit ($M$), we have $D \cdot (S_{\text{syn}} + S_{\text{entropy}}) \le M$. Rearranging this inequality yields $D \le \frac{M}{S_{\text{syn}} + S_{\text{entropy}}}$. By Corollary~1, successful entropy-based evasion requires $S_{\mathrm{entropy}} \ge w$. Substituting this yields Eq. \eqref{eq:entropy_depth_bound}. Thus, the attacker cannot simultaneously maximize both evasion entropy and recursion depth. \hfill $\blacksquare$

\paragraph{Application to ETSI Security V2 and B-SSRF}
Theorem 1 establishes a theoretical capacity limit independent of the implementation. To understand its impact on our proposed defense, we analyze the specific protocol and detector constraints. In the ETSI C-ITS Security V2 specification, the OER recursive structure dictates at least $S_{\text{syn}} = 2$ bytes per layer (one byte for the \texttt{Choice} index and one byte for the \texttt{Length} determinant).

For the detector configuration used throughout this work, the sliding window size is fixed to $w=64$. Consequently, Corollary~1 requires an adversary attempting entropy-based evasion to introduce at least 64 bytes of random padding between consecutive recursive markers. Substituting $M=1400$, $S_{\text{syn}}=2$, and $w=64$ into Theorem~1 yields:
\[
D_{\max} = \left\lfloor \frac{1400}{2 + S_{\mathrm{entropy}}} \right\rfloor .
\]
For representative padding budgets:
\[
\begin{aligned}1S_{\mathrm{entropy}} = 0\text{ B} &\Rightarrow D_{\max} = 700 \\
S_{\mathrm{entropy}} = 32\text{ B} &\Rightarrow D_{\max} = 41 \\
S_{\mathrm{entropy}} = 64\text{ B} &\Rightarrow D_{\max} = 21 \\
S_{\mathrm{entropy}} = 128\text{ B} &\Rightarrow D_{\max} = 11.
\end{aligned}
\]
Under our detector calibration ($w = 64$), the operational upper bound on the exploit depth is restricted to $D_{\max} \le 21$. This mathematical trade-off is visualized in Figure \ref{fig:pareto_curve}.

\paragraph{Empirical Latency Impact}
Theorem 1 implies that successful entropy-based evasion necessarily reduces the achievable recursion depth. Accordingly, our experiments observe that an OER structure nested to $D = 21$ incurs a nominal parsing latency of approximately $15.4\,\mu\text{s}$. Compared to the unmitigated stack-exhaustion latency of $960.41\,\mu\text{s}$ at $D = 668$ (detailed in Section VI-A), this represents a 98.4\% reduction in CPU overhead. Consequently, the physical resource bound defined by Theorem 1 significantly constrains the adversary's ability to jointly maximize evasion entropy and recursive depth, significantly limiting the workload amplification threat.

\subsection{Probabilistic Risk Budget FSM (PRB-FSM)}

To prevent resource exhaustion without inflating baseline jitter, the PRB-FSM adapts the inspection rate based on a continuous risk budget $B_t \in [0, 100]$ (Figure \ref{fig:prb_fsm_dynamics}), drawing inspiration from adaptive monitoring \cite{Sosnowski2024} % TODO: Verify Sosnowski 2021 adaptive traffic security monitoring paper
and software circuit breakers \cite{Nygard2007}. % TODO: Verify Nygard 2007 software circuit breakers (Release It!) reference
The budget update is defined as:
\begin{equation}
B_{t+1} = \min(B_{max}, \max(0, B_t - \mathcal{P}_t + \gamma_{streak}))
\end{equation}
where $\mathcal{P}_t$ is the anomaly penalty and $\gamma_{streak}$ is the adaptive recovery rate. Under sustained streaks of clean packets, the recovery rate $\gamma_{streak}$ is accelerated (e.g., $6\times$) to restore the budget and return to nominal low-overhead sampling once the threat dissipates.

Budget thresholds $\{\tau_1, \tau_2, \tau_3\}$ map the continuous budget to discrete operational states ($S_0 \dots S_3$) with escalating sampling rates (10\%, 50\%, 100\%). This ensures that nominal operations incur minimal overhead ($\sim$10\% inspection) while providing rapid, risk-aware escalation under pressure.

Coupling the B-SSRF with the probabilistic risk budget reduces the amortized computational overhead. Let $p_t \in [0.1, 1.0]$ be the dynamic inspection probability. The expected inspection cost $\mathbb{E}[C_{\text{inspect}}]$ per ingress packet is bounded by:
\begin{equation}
\mathbb{E}[C_{\text{inspect}}] = p_t \cdot \mathcal{O}(L_{\max}) + (1 - p_t) \cdot \mathcal{O}(1_{\text{routing}}) \approx \mathcal{O}(1)
\end{equation}
This formulation establishes a deterministic security floor. To safeguard the system against exploratory policy divergence or algorithmic degradation during Deep Q-Network (DQN) optimization, a Layer-1 Overrun Guard is hardcoded in the parser kernel: if the transient risk budget $B_t$ depletes past a severe threat threshold ($\tau_2$), the C++ state machine overrides the active DRL policy and forces the packet sampling probability to $p_t = 1.0$ (100\% telemetry inspection). This heuristic override guarantees that the computational exposure lower bound remains strictly bounded under sustained cascading exploits.

To balance processing overhead and security under non-stationary workloads, we formulate parameter modulation as a Markov Decision Process (MDP \cite{SuttonBarto2018}). At each control window $t$, the node monitors queuing and computing telemetry to form a state vector $s_t$, selects parameter adjustments $a_t$ via DQN \cite{mnih_human-level_2015}, and receives a Lagrangian-constrained reward $r_t$ designed to minimize latency overhead while enforcing strict security boundaries.

\paragraph{Observation Space} The state vector $s_t \in \mathbb{R}^3$ captures the instantaneous operational environment of the V2X stack: 
$$s_t = \begin{bmatrix} o_{\text{rate}} \\ o_{\text{sq}} \\ o_{\text{anomaly}} \end{bmatrix}$$
where:
\begin{enumerate}
    \item $o_{\text{rate}}$ is the current baseline FSM packet sampling rate ($p_t \in [0.05, 1.0]$).
    \item $o_{\text{sq}}$ is the normalized window-averaged Sum-of-Squares similarity value, computed as $o_{\text{sq}} = Q_{\text{avg}} / Q_{\text{max}}$, where $Q_{\text{max}} = 4096.0$ is the theoretical maximum variance boundary of the sliding window.
    \item $o_{\text{anomaly}}$ is the observed malware/exploit ratio within the control window ($N_{\text{anomaly}} / N_{\text{total}}$).
\end{enumerate}
This low-dimensional state representation minimizes telemetry calculation overhead at the C++ data plane.

\paragraph{Action Space} The action space is modeled as a discrete set of rate control decisions. The DQN agent selects an action index $a_t \in \mathcal{A} = \{0, 1, 2, 3, 4\}$, which maps to a relative change in the FSM base sampling rate $\Delta p \in \mathcal{M} = \{-0.10, -0.05, 0.0, 0.05, 0.10\}$. The base sampling rate for the next epoch is updated and clamped via:
\begin{equation}
p_{t+1} = \max\left(0.05, \min\left(1.0, p_t + \Delta p\right)\right)
\end{equation}
Other FSM parameters (recovery rate $R_{\text{rec}} = 0.05$, penalty factor $K_{\text{pen}} = 50.0$, and threshold $\theta_{\text{sq}} = 600$) are maintained at nominal baseline configurations. This discrete step transition bounds policy exploration within safe operational margins. Integrally, the DRL-controlled base sampling rate $p_t$ represents the active sampling rate strictly during the nominal state ($S_0$) of the PRB-FSM. When anomalous workload depletion triggers state transitions ($S_1$ or $S_2$), the Layer-1 Overrun Guard (heuristic rules of the state machine) temporarily overrides the active DRL base rate, forcing the sampling probability to the deterministic configurations of $0.5$ or $1.0$ respectively until the budget recovers.

\paragraph{Lagrangian-Constrained Reward Function} To drive policy convergence toward V2X QoS preservation, we formulate reward shaping as a constrained optimization problem. The agent's goal is to minimize inspection overhead subject to a maximum tolerable exploit leakage target $\epsilon_{\text{target}} = 0.01$ (1\%):
\begin{equation}
\begin{aligned}
\min_{a} \quad & \mathbb{E} \left[ c_{\text{overhead}} \cdot p_{\text{inspect}} \right] \\
\text{s.t.} \quad & R_{\text{leakage}} \le \epsilon_{\text{target}}
\end{aligned}
\end{equation}
Using the Lagrangian relaxation framework, the step reward $R_t$ is defined as:
\begin{equation}
R_t = - c_{\text{overhead}} \cdot p_{\text{inspect}} - \lambda_t \cdot \max\left(0, R_{\text{leakage}} - \epsilon_{\text{target}}\right)
\end{equation}
where $c_{\text{overhead}} = 2.0$ represents the overhead scale factor, $R_{\text{leakage}} = FN / (FN + TP)$ is the packet leakage rate (false negatives divided by total true malware), and $\lambda_t$ is the dynamic Lagrangian multiplier. During online interactive training, $\lambda_t$ is updated at each epoch to dynamically scale the penalty weight based on current leakage violations:
\begin{equation}
\lambda_{t+1} = \max\left(0, \lambda_t + \alpha \cdot \left(R_{\text{leakage\_avg}} - \epsilon_{\text{target}}\right)\right)
\end{equation}
where $\alpha = 0.05$ is the constraint gradient learning rate. This adaptive formulation automatically penalizes high leakage rates with massive budget restrictions while allowing the model to relax sampling rates under peace times.

\subsection{Digital Twin Co-simulation Framework}
To bridge the C++ V2X simulation kernel with the MLOps optimization framework, we establish a lightweight, decoupled digital twin \cite{Digital_Twin} architecture, which computationally mirrors the physical vehicular data-path to enable safe, offline policy training and validation before real-world deployment. The C++ simulation kernel acts as the physical edge environment, while the cognitive Python daemon processes telemetry dynamically.

\paragraph{Zero-Allocation IPC Protocol} To avoid serialization bottlenecks (e.g., gRPC), the kernel serializes metrics into a 40-byte fixed-size structure:
\begin{equation}
\mathcal{B}_{\text{telemetry}} = \langle \text{TP}, \text{TN}, \text{FP}, \text{FN}, N_{\text{inspected}}, Q_{\text{sq}}, T_{\text{latency}}, S_{\text{rate}} \rangle
\end{equation}
Raw memory offsets allow $\mathcal{O}(1)$ binary decoding under sub-microsecond latency.

\paragraph{Lightweight Edge Inference via ONNX} To guarantee deployment viability on resource-constrained MEC nodes, the converged PyTorch policy is exported to an Open Neural Network Exchange (ONNX) graph. By executing the DRL forward-pass directly through the C++ ONNX Runtime API, we eliminate the Python Global Interpreter Lock (GIL) overhead, achieving deterministic microsecond inference.

\section{Experimental Setup}

\subsection{Experimental Testbed and Traffic Modalities}
The testbed utilizes a 12th Gen Intel i7-12700H and 16 GB of RAM running Ubuntu 22.04 LTS. To accurately simulate the resource constraints of a typical MEC edge server or Roadside Unit (RSU) rather than high-performance parallel computing hosts, all C++ simulation and Python daemon processes are pinned to a single physical CPU core (using \texttt{taskset}). The software is based on Vanetza \cite{Vanetza}, an open-source ETSI C-ITS suite processing CAM/DENM payloads via the \texttt{asn1c} compiler \cite{asn1c_compiler}. We integrated a custom QoS evaluation framework into the ingress pipeline to measure presentation-layer latency. Crucially, although the evaluation runs on an x86 platform, the $\mathcal{O}(1)$ algorithmic complexity and zero-allocation memory design of the B-SSRF mathematically guarantee that its relative efficiency gains and tail-latency reduction ratios (e.g., the 85.5\% reduction) scale proportionally to low-power, ARM-based AIoT edge devices. 

We interleaved compliant CAM/DENM traffic with malicious recursion payloads across multi-scale injection rates ranging from 0.1\% to 10\% (specifically 0.1\%, 0.5\%, 1.0\%, 5.0\%, and 10.0\%). To guarantee reproducibility, we evaluated two distinct baselines tracked by git commit hashes: (1) the \textbf{Unpatched Baseline} (\texttt{3a18193}), representing the native vulnerable parser; and (2) the \textbf{Patched Baseline} (\texttt{2daa68f}), representing the official static recursion limit. The system was evaluated under four scheduling configurations: Mode 0 (Continuous Uniform), Mode 1 (Single Strong Pulse), Mode 2 (Periodic Pulsing).

\subsection{Metrics and Calibration}
We micro-benchmark parser execution latency per packet. Across millions of payloads, we measure the Mean, Median, P99, and Max Latency to quantify queue delay and Head-of-Line (HoL) blocking. To handle sparse 1\% threats, we calibrated the PRB-FSM thresholds with a clean streak threshold of 1,000 packets and a 10\% base sampling rate. Measurements were aggregated over 30 runs to marginalize OS scheduler noise.

\subsection{Adaptive Shedding \& First-Miss Tolerance}
Probabilistic sampling systems inherently face a "First-Miss Penalty," producing the transient Max Latency spikes (e.g., $\sim$4.5 ms) observed in Table \ref{tab:master}. To safeguard the edge node against exploratory policy divergence or algorithmic degradation of the AI framework, our architecture enforces a \textbf{two-tier deterministic guarantee}. A hardcoded \textbf{Layer-1 Overrun Guard (heuristic FSM)} is embedded in the C++ parser kernel; if the transient risk budget depletes past a severe threat threshold, the C++ state machine immediately \textbf{overrides the active DRL policy} and forces the packet sampling probability to 1.0 (100\% full telemetry inspection). 

Within this hard safety ceiling, the DQN agent (converging reliably during co-simulation, Fig. \ref{fig:rl_convergence}) learns a Lagrangian-constrained policy that dynamically optimizes the sampling risk budget during nominal operations. Upon detecting structural instability, the PRB-FSM instantaneously transitions to the quarantine state, strictly bounding the HoL impact. Furthermore, as illustrated in Fig. \ref{fig:periodic_timeline} and \ref{fig:pulse_timeline}, the accelerated recovery mechanism ensures millisecond-level budget replenishment once threats dissipate, preventing long-term inspection overhead lock-in and minimizing the host CPU footprint during peace times.

A robust AIoT edge defense demands an optimal balance between latency stability and detection accuracy. Traditional deep packet inspection (DPI) introduces constant parsing overhead and severe jitter, which is unacceptable for safety-critical V2X applications. In contrast, our digital twin framework utilizes a DRL-driven FSM to dynamically adapt probabilistic sampling. During nominal periods ($S_0$), the system operates at a minimal base sampling rate, restricting processing complexity to $\mathcal{O}(R_{\text{sample}} \times w)$. By exploiting the high entropy of legitimate signed OER messages, this self-adaptive stance achieves a 0.00\% false positive rate (FPR) across all evaluated modalities (Table~\ref{tab:fpr_fnr}), preserving cooperative awareness without disrupting standard vehicular communications.

The empirical accuracy profile reveals an intrinsic systems-security trade-off under sparse threat models. As documented in Table~\ref{tab:fpr_fnr}, the false negative rate (FNR) scales down as the attack intensity increases. Under high-intensity workloads ($\ge 1.0\%$), the filter maintains a tight FNR boundary ($\le 0.11\%$). This variance represents a classic "First-Miss Penalty" before the transient risk budget depletes past the quarantine threshold. From an AIoT resource optimization perspective, this is an intentional, cost-effective trade-off: the system actively tolerates minor initial leakages to protect edge CPU resources from continuous inspection fatigue during low-threat windows, relying on static application-layer limits to catch residual bypassed frames.

Faced with stateful adversaries executing low-frequency pulsing exploits to induce persistent inspection lock-in, the framework maintains robust systemic containment. Because the underlying B-SSRF operates with strict $\mathcal{O}(1)$ complexity, forcing the data plane into continuous inspection shifts the baseline processing time negligibly without triggering localized Head-of-Line blocking. The zero-allocation IPC and decoupled digital twin design ensure that the runtime execution path remains deterministic. Policy validity is rigorously enforced by Lagrangian reward shaping rather than fragile heuristic parameter tuning, demonstrating highly generalized operational resilience across heterogeneous, resource-constrained V2X infrastructures.

\section{Limitations and Future Work}

While our two-tier framework successfully establishes a deterministic physical bound on presentation-layer resource exhaustion, the current evaluation isolates the dynamic mitigation to a single MEC edge node. In dense Cooperative Intelligent Transport Systems (C-ITS), vehicular mobility and fluctuating traffic densities dynamically alter the spatial distribution of structural threats. Future research must expand the closed-loop digital twin into a multi-node federated architecture (e.g., via ns-3 network-level simulations). By sharing the DRL agent's episodic rewards and risk budget policies across distributed Roadside Units (RSUs), the network can proactively establish intersection-wide threat containment and cooperative load-shedding before localized HoL blocking cascades into neighboring zones.

Furthermore, although our lightweight ONNX integration \cite{ONNX} (compiled from PyTorch models \cite{Pytorch}) successfully eliminates the Python GIL bottleneck to execute AI inference in microseconds, the $\mathcal{O}(1)$ B-SSRF pre-filter still consumes nominal host CPU cycles during L2/L3 memory access. For ultra-low-latency AIoT deployments, migrating this fixed-parameter structural sketch array directly to Programmable Data Planes (PDP)—such as SmartNICs via P4 \cite{SmartNIC_2019} or eBPF/XDP kernels \cite{Sadiq_DataPlane_2023}—presents a highly promising avenue. By pushing the $F_2$ frequency extraction entirely to the network interface card, future iterations can achieve true zero-host-CPU mitigation, fully decoupling security telemetry from vehicular application processing.

\section{Conclusion}

The deep integration of MEC into V2X networks introduces critical presentation-layer vulnerabilities. Under legacy ASN.1/OER specifications, computationally bounded attackers can inject MTU-constrained structural recursion bombs that rapidly exhaust edge parser resources, inducing severe Head-of-Line (HoL) blocking and violating sub-10 ms safety constraints. 

To secure resource-constrained AIoT edge infrastructure without resorting to cost-prohibitive firmware upgrades or heavy continuous inspection, this paper presented a DRL-driven Parser-Layer Circuit Breaker. By decoupling a fast-path C++ data plane from a cognitive Python slow path, we engineered a closed-loop digital twin bridged by zero-allocation IPC. At its core, the architecture enforces a rigorous two-tier deterministic guarantee: an $\mathcal{O}(1)$ B-SSRF utilizes fixed mathematical boundaries ($w=64$) to lock in a heuristic probabilistic risk budget, which in turn acts as a hard Layer-1 Overrun Guard to constrain the Deep Q-Network's exploratory policy. 

Extensive empirical evaluations demonstrated that our dynamic sampling mitigation reduces P99 tail latency by 85.5\% (from 0.3478 ms down to 0.0503 ms) under high-intensity attacks. By transforming the security objective from static payload signature matching into dynamic computational exposure management, our framework delivers microsecond-level adaptive threat containment, successfully preserving critical V2X cooperative awareness under intense adversarial workloads.
