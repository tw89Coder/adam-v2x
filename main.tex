\begin{abstract}
Vehicle-to-Everything (V2X) communications rely on low-latency presentation-layer processing. However, Abstract Syntax Notation One (ASN.1) parsers utilizing Octet Encoding Rules (OER) are vulnerable to algorithmic complexity attacks (CWE-674). We characterize this workload amplification under strict Maximum Transmission Unit (MTU) constraints, showing that a payload only 8.6\% larger than a legitimate 325-byte message induces a 5.7$\times$ latency amplification, causing Head-of-Line (HoL) blocking. To satisfy strict V2X real-time constraints, we evaluate an ultra-lightweight mitigation strategy combining a constant-memory Streaming Structural Risk Filter (SSRF) and a Deep Reinforcement Learning (DRL)-driven Probabilistic Risk Budget Finite State Machine (PRB-FSM). By shifting the defense to dynamic, DRL-optimized probabilistic sampling, our approach achieves negligible overhead during nominal operations and rapid mitigation under attack. Evaluation shows the mitigation reduces P99 tail latency from 0.1793 ms to 0.0320 ms, containing the amplification surface.
\end{abstract}

\end{IEEEkeywords}

\section{Introduction}

Cooperative Intelligent Transport Systems (C-ITS) rely on Vehicle-to-Everything (V2X) communication to support safety-critical applications requiring latency below 10 ms \cite{3GPP_URLLC}. % TODO: Verify 3GPP V2X latency requirement standard (TR 38.913)
While Multi-access Edge Computing (MEC) offloads complex tasks \cite{ETSI_MEC}, % TODO: Verify ETSI MEC framework and reference architecture (GS MEC 003)
presentation-layer parsing of messages like Cooperative Awareness Messages (CAM) \cite{ETSI_CAM} % TODO: Verify ETSI CAM specification (EN 302 637-2)
and Decentralized Environmental Notification Messages (DENM) \cite{ETSI_DENM} % TODO: Verify ETSI DENM specification (EN 302 637-3)
remains a potential security bottleneck. These payloads are defined via Abstract Syntax Notation One (ASN.1) \cite{ETSI_ASN1} % TODO: Verify ETSI Facilities layer common data dictionary (TS 102 894-2)
and encoded using Octet Encoding Rules (OER) \cite{ITU_OER} % TODO: Verify ITU-T X.696 OER encoding rules standard
for processing efficiency.

However, OER's support for recursive structures enables algorithmic complexity attacks (CWE-674). By crafting payloads with deeply nested choice fields, an attacker can trigger deep recursion or exponential object tree expansion. In edge workloads, these "structural bombs" exhaust CPU resources and cause Head-of-Line (HoL) blocking, inducing severe processing latency spikes for queued legitimate safety messages.

Conventional defenses introduce continuous inspection overhead that degrades system jitter and inflates P99 tail latency under high-volume traffic. To resolve the conflict between security verification and low-latency packet forwarding, we evaluate a \textit{Parser-Layer Circuit Breaker for Workload Mitigation} (Figure \ref{fig:architecture}). Our approach combines a constant-memory Streaming Structural Risk Filter (SSRF) with a Probabilistic Risk Budget FSM (PRB-FSM) driven by a Deep Reinforcement Learning (DRL) agent. By shifting the security objective to \textbf{mitigating computational exposure}, we contain adversarial workloads before they reach the parser, dynamically modulating inspection intensity to preserve QoS.

\section{Background and Threat Model}

\subsection{Background: V2X, ASN.1, and OER}

V2X systems rely on the Facilities layer to generate and parse standardized messages (e.g., CAM, DENM) defined in ASN.1 and encoded using Octet Encoding Rules (OER) to achieve high performance in vehicular environments.

MEC edge servers handle presentation-layer parsing of ASN.1 payloads. OER recursive structures are common in V2X certificates and security headers (e.g., IEEE 1609.2 \cite{IEEE_1609_2}), % TODO: Verify IEEE 1609.2 secure vehicular communication standard
allowing for deeply nested object trees. Under CWE-674 \cite{CWE_674} % TODO: Verify CWE-674 (Uncontrolled Recursion) vulnerability definition
attacks, an adversary exploits these definitions to trigger parser-level stack exhaustion, forcing thousands of recursive allocations from a small payload.

This manifests as Head-of-Line (HoL) blocking in edge pipelines, where legitimate, time-sensitive safety messages queued behind a malicious payload suffer severe latency jitter even without volumetric saturation.

We assume a computationally bounded adversary $\mathcal{A}$ capable of injecting syntactically valid but structurally malicious OER payloads (CWE-674 recursion bombs). To achieve deep nesting within MTU limits, the attacker must inject repetitive structural markers (e.g., identical choice indices), which forms the basis of our detection. The adversary's goal is to maximize parsing delay $L = L_{queue} + C_{parse} + C_{inspection}$ to trigger HoL blocking and degrade QoS.

The attack targets parser resource exhaustion \textit{prior} to semantic validation (e.g., MDS \cite{vanderHeijden2018}), % TODO: Verify van der Heijden 2018 survey on misbehavior detection in C-ITS
exploiting the parser during IEEE 1609.2 certificate decoding before signature verification is completed. Assuming a latency budget (e.g., $L \le 10$ ms \cite{3GPP_URLLC}), % TODO: Verify 3GPP V2X latency requirement standard (TR 38.913)
our security goal is to bound the pre-parsing inspection overhead to $C_{inspection} \le \mathcal{O}(1)$ relative to the payload size, mitigating workload escalation.

To maintain a focused research scope, general volumetric DDoS attacks (saturating network bandwidth), cryptographic compromise (e.g., ECDSA cryptanalysis), replay/Sybil attacks (handled by security envelope/MDS layers), and PKI/CA trust compromise are considered out-of-scope.

\subsection{Parser Workload Amplification Analysis}

Algorithmic complexity attacks exploit the predictable expansion of encoded bit-streams. In OER, recursive definitions (e.g., nested choice fields) force repeated memory allocations and pointer dereferences.

An OER "recursion bomb" maximizes nesting depth. Since V2X enforces strict MTU limits, achieving deep recursion within a small footprint requires repetitive structural markers (e.g., choice indices). This structural repetition enables severe computational amplification, allowing a payload of a few hundred bytes to exhaust CPU cycles.

Our workloads reveal that recursion bombs exhibit long sequences of identical octets (e.g., \texttt{0x02}), contrasting with high-entropy cryptographic fields in legitimate traffic. Our SSRF identifies this structural repetition without semantic parsing.

This parsing vulnerability leads to remote denial-of-service (CWE-248 \cite{CWE_248}, % TODO: Verify CWE-248 (Unhandled Exception) vulnerability definition
CVE-2026-37554 \cite{CVE_2026_37554}), % TODO: Verify CVE-2026-37554 recursion stack overflow exploit details
where MTU-limited payloads trigger stack overflows and cause up to a 24.52$\times$ processing latency amplification. This motivates our DRL-driven dynamic mitigation framework.

\section{Bounded Risk-Aware Pre-Parsing Defense (BRPPD)}

To balance extreme security against CWE-674 structural attacks with the stringent low-latency constraints of V2X communications, we propose a bounded-cost pre-parsing framework. The system provides bounded-cost inspection prior to ASN.1 decoding.

\subsection{Bounded Streaming Structural Risk Filter (B-SSRF)}
We calculate the localized second frequency moment ($F_2$) of byte frequencies within a sliding window to estimate structural instability. This metric, denoted as $SQ$, serves as a collision concentration statistic:
$$SQ = \sum_{j \in \mathcal{V}} c_j^2$$
where $c_j$ is the frequency of byte value $j$ in the window. For legitimate high-entropy V2X payloads, $c_j$ is dispersed, resulting in a low $SQ$. Conversely, OER recursion bombs rely on concentrated structural markers (e.g., choice indices) within MTU constraints, inflating the $F_2$ moment. This is commonly used in streaming algorithms to estimate collision concentration in bounded memory \cite{Alon1996, Krishnamurthy2003}. % TODO: Verify Alon 1996 (AMS sketch) and Krishnamurthy 2003 sketch-based change detection papers
Repeated structural expansion markers in recursive payloads cause the local symbol distribution to concentrate, significantly increasing $SQ$ compared to standard CAM/DENM traffic.

Algorithm \ref{alg:bssrf} defines this bounded filter. Importantly, the correctness and bounding properties of the framework rely strictly on the mathematical properties of the $F_2$ estimator, avoiding brittle, protocol-specific signatures. By evaluating localized structural variance rather than static patterns, the detector maintains generalizability against evasive MTU-compliant recursion bombs.

\begin{algorithm}[!ht]
\caption{Bounded Streaming Structural Risk Filter (B-SSRF)}
\label{alg:bssrf}
\begin{algorithmic}[1]
\Require Payload $P$, window size $w$, scanning limit $L_{max}$, threshold $\theta$
\State Initialize frequency sketch $H[256] \gets 0$
\State Initialize sliding window $W$
\State $SQ \gets 0$ \Comment{Localized $F_2$ moment}
\State $n_{scan} \gets \min(|P|, L_{max})$

\For{$i = 0$ to $n_{scan} - 1$}
    \State $b \gets P[i]$
    \If{$|W| = w$}
        \State $b_{old} \gets W.\text{pop\_front}()$
        \State $SQ \gets SQ - (2H[b_{old}] - 1)$
        \State $H[b_{old}] \gets H[b_{old}] - 1$
    \EndIf
    \State $W.\text{push\_back}(b)$
    \State $H[b] \gets H[b] + 1$
    \State $SQ \gets SQ + (2H[b] - 1)$ \Comment{Amortized $F_2$ update}

    \If{$|W| = w$ and $SQ > \theta$}
        \State \Return \textbf{True} \Comment{Structural anomaly detected}
    \EndIf
\EndFor
\State \Return \textbf{False}
\end{algorithmic}
\end{algorithm}

This filter bounds the per-packet processing complexity to $\mathcal{O}(w)$ for a constant window size $w$. By restricting the scan to $L_{max}$ bytes, the filter overhead is strictly $\mathcal{O}(1)$ relative to the packet length $N$, consuming negligible CPU cycles compared to standard ASN.1 object allocation.

\subsection{Probabilistic Risk Budget FSM (PRB-FSM)}

To prevent resource exhaustion without inflating baseline jitter, the PRB-FSM adapts the inspection rate based on a continuous risk budget $B_t \in [0, 100]$ (Figure \ref{fig:prb_fsm_dynamics}), drawing inspiration from adaptive monitoring \cite{Sosnowski2021} % TODO: Verify Sosnowski 2021 adaptive traffic security monitoring paper
and software circuit breakers \cite{Nygard2007}. % TODO: Verify Nygard 2007 software circuit breakers (Release It!) reference
The budget update is defined as:
$$B_{t+1} = \min(B_{max}, \max(0, B_t - \mathcal{P}_t + \gamma_{streak}))$$
where $\mathcal{P}_t$ is the anomaly penalty and $\gamma_{streak}$ is the adaptive recovery rate. Under sustained streaks of clean packets, the recovery rate $\gamma_{streak}$ is accelerated (e.g., $6\times$) to restore the budget and return to nominal low-overhead sampling once the threat dissipates.

Budget thresholds $\{\tau_1, \tau_2, \tau_3\}$ map the continuous budget to discrete operational states ($S_0 \dots S_3$) with escalating sampling rates (10\%, 50\%, 100\%). This ensures that nominal operations incur minimal overhead ($\sim$10\% inspection) while providing rapid, risk-aware escalation under pressure.

Coupling the B-SSRF with the risk budget bounds the inspection cost $C_{inspection}(P_i)$:
$$C_{inspection}(P_i) = \begin{cases} \mathcal{O}(1) & R_i \le \theta \\ \mathcal{O}(k) & R_i > \theta, \; k \ll N \end{cases}$$
This formulation establishes a deterministic security floor. To safeguard the system against exploratory policy divergence or algorithmic degradation in both DQN and PPO paradigms, a Layer-1 Overrun Guard is hardcoded in the parser kernel: if the transient risk budget $B_t$ depletes past a severe threat threshold ($\tau_2$), the C++ state machine overrides the active DRL policy and forces the package sampling probability to $p_t = 1.0$ (100\% telemetry inspection). This heuristic override guarantees that the computational exposure lower bound remains strictly bounded under sustained cascading exploits.

To balance processing overhead and security under non-stationary workloads, we formulate parameter modulation as a Markov Decision Process (MDP \cite{SuttonBarto2018}). % TODO: Verify Sutton & Barto 2018 reinforcement learning textbook
At each epoch $t$, the node monitors queuing and computing telemetry to form a state vector $s_t$, selects parameter adjustments $a_t$ via DRL \cite{Mnih2015_Nature}, % TODO: Verify Mnih 2015 Nature human-level control (DQN) paper
and receives a multi-objective reward $r_t$ minimizing latency jitter and maximizing threat containment.

\paragraph{State Space} The state space $S_t \subset \mathbb{R}^5$ captures the operational environment and transient workload characteristics of the V2X stack: $s_t = \begin{bmatrix} SQ_t & \lambda_t & \bar{N}_t & Q_t & U_t \end{bmatrix}^T$, where $SQ_t$ is the localized collision concentration estimated by the SSRF sliding window, $\lambda_t$ is the packet arrival rate, $\bar{N}_t$ is the average packet size, $Q_t$ is the ingress buffer queue length, and $U_t$ represents parsing thread core CPU utilization. To ensure stable neural network convergence and mitigate magnitude disparities among heterogeneous metrics (e.g., $\lambda_t$ vs. $U_t$), each feature component in $s_t$ is dynamically normalized to $[0, 1]$ using a min-max scaler scaled by historical telemetry boundaries.

\paragraph{Action Space} To optimize parameter settings, the action space $A_t$ is primarily formulated as a discrete action space for Deep Q-Networks (DQN \cite{Mnih2015_Nature}). % TODO: Verify Mnih 2015 Nature human-level control (DQN) paper
The discrete action $a_t \in \mathcal{A}_{disc} = \{0, 1, \dots, K-1\}$ maps to a discrete grid of transition budgets, penalties $\mathcal{P}_t$, state thresholds, and base sampling rates. Alternatively, to support continuous action spaces (such as Proximal Policy Optimization, PPO \cite{Schulman2017_PPO}), % TODO: Verify Schulman 2017 PPO paper
the action vector can be represented as $a_t = \begin{bmatrix} \mathcal{P}_t & \theta_t & p_t \end{bmatrix}^T$ to directly adjust the risk budget penalty weight, the detection threshold $\theta$, and the sampling probability $p_t \in [0.1, 1.0]$. To constrain policy exploration within a deterministic safe sandbox, a Layer-2 Heuristic Clamping interface filters all agent directives before deployment. For continuous PPO actions, out-of-bound variables are clipped via action spaces mapping to hardware limits, while discrete DQN decisions pass through a boundary translator enforcing a baseline rate adjustment: $p_{next} = \max(0.05, \min(1.0, p_{curr} + \Delta p))$. If anomalous outputs breach empirical safety thresholds (e.g., setting the structural threshold $\theta > 650$ or penalty scaling below minimum boundaries), the native runtime dynamically clamps variables to verified operational margins, maintaining an inviolable defense-in-depth lower bound.

\paragraph{Multi-Objective Reward Function} To drive policy convergence toward QoS preservation and threat containment, the reward function $R_t$ is formulated as a multi-objective weighted sum:
\begin{equation}
R_t = - \left( w_1 \cdot \mathcal{L}_t + w_2 \cdot \mathcal{D}_t + w_3 \cdot \mathcal{O}_t \right)
\end{equation}
where $\mathcal{L}_t \in \{0, 1\}$ is a binary indicator of security leakage (an anomalous packet bypassing the filter and reaching the parser), $\mathcal{D}_t = \max(0, L_{P99} - L_{target})$ penalizes queue latency when P99 tail latency exceeds the target safety budget (e.g., $L_{target} = 0.1$ ms), and $\mathcal{O}_t = p_t \cdot C_{inspect}$ represents computational inspection overhead. The weights $w_1, w_2, w_3$ are specified in the policy hyperparameters.


\section{Experimental Setup}

To evaluate the effectiveness and performance overhead of the proposed Parser-Layer Circuit Breaker, we developed a controlled evaluation framework built around the Vanetza-based test bench, emulating a high-throughput presentation-layer workload.

\subsection{Experimental Testbed and Traffic Modalities}
The testbed utilizes a 12th Gen Intel i7-12700H and 16 GB of RAM running Ubuntu 22.04 LTS. The software is based on Vanetza \cite{Vanetza}, % TODO: Verify Riebl Vanetza open-source ETSI C-ITS repository citation
an open-source ETSI C-ITS suite processing CAM/DENM payloads via the \texttt{asn1c} compiler \cite{asn1c_compiler}. % TODO: Verify Walkin asn1c compiler documentation/repository
We integrated a custom QoS evaluation framework into the Vanetza ingress pipeline to measure presentation-layer latency. The $\mathcal{O}(N)$ workload amplification and its $\mathcal{O}(1)$ mitigation are architecturally agnostic.

We interleaved compliant CAM/DENM traffic from the Vanetza corpus with malicious IEEE 1609.2 recursion payloads at 1\%, 5\%, and 10\% injection rates. We evaluated three configurations: \textbf{Mode 0 (Unpatched Native)} baseline vulnerable parser; \textbf{Mode 1 (Official Patch)} using static \texttt{asn1c} recursion limits; and \textbf{Mode 2 (Hybrid Defense)} combining Mode 1 with our proposed Adaptive Circuit Breaker FSM at the network edge.

\subsection{Metrics, Calibration, and Rigor}
We focus on micro-benchmarking parser execution latency per packet. Across millions of processed payloads, we measure the Mean, Median, P99/P99.9 Tail, and Max Latency to quantify queue delay and Head-of-Line blocking, assessing jitter via tail-latency variance.

To handle sparse 1\% threats, we calibrated the PRB-FSM thresholds $\{\tau_1, \tau_2, \tau_3\}$ with a clean streak threshold of 1,000 packets and a 10\% base sampling rate. This ensures the system remains in elevated risk states during intermittent attack gaps, clamping the FNR to $<0.01\%$ without rigid signatures.

Measurements were aggregated over 30 runs to marginalize scheduler noise. MEC suitability is demonstrated by a minimal memory footprint (256-entry frequency sketch) and latency stability under a 10\% attack load. To facilitate reproducibility, we utilized fixed random seeds and specific software revision hashes of the baseline repository, denoting the vulnerable base (\texttt{3a18193}), the patched base (\texttt{2daa68f}), and the latest reference base (\texttt{575fe31}). The complete evaluation framework and source code are open-source and publicly available at \url{https://github.com/tw89Coder/v2x-asn1-drl}.

\subsection{Digital Twin Co-simulation Framework}
To bridge the C++ V2X simulation kernel with the MLOps optimization framework, we establish a closed-loop \textbf{Digital Twin} \cite{Digital_Twin} % TODO: Verify Fuller 2020 Digital Twin enabling technologies survey paper
architecture. The C++ simulation kernel acts as the physical edge environment, while the cognitive daemon serves as its digital twin, processing telemetry dynamically.

\paragraph{Zero-Allocation Wire Protocol} To avoid serialization overhead, we implement a Zero-Allocation Binary Struct Protocol. The simulation kernel serializes metrics into a 40-byte fixed-size structure:
\begin{equation}
\mathcal{B}_{telemetry} = \langle \text{Seq}, \text{Freq}, \bar{N}, Q, U, SQ, \Delta t, \mathcal{L} \rangle
\end{equation}
where fields include counts, window moments, and performance metrics. Raw memory offsets allow $O(1)$ binary decoding under sub-microsecond serialization latency.

\paragraph{Training and Thread Boundaries} To prevent training pauses from blocking packet queues, the optimization interface isolates execution into a low-latency socket listener thread and an asynchronous model updating thread.

\paragraph{Dynamic Scaling} PyTorch \cite{Pytorch} % TODO: Verify Paszke 2019 PyTorch framework paper
neural layers auto-scale dynamically by querying active features ($state\_dim$) and action mappings ($action\_dim$) directly from configuration files, allowing seamless scaling without architectural changes.

To validate the efficacy of the Bounded Streaming Structural Risk Filter (B-SSRF), we empirically evaluated the Localized Second Frequency Moment ($F_2$, denoted as $SQ$) across our dataset using a 64-byte sliding window. For legitimate V2X traffic, such as a standard 325-byte CAM payload, the presence of high-entropy cryptographic fields (e.g., ECC signatures and certificates) naturally disperses the byte distribution. Consequently, the localized collision concentration remains low. In our empirical evaluation, the legitimate payloads exhibited a peak $SQ$ of 286, demonstrating the high structural variance inherent to standard protocol operations.

In stark contrast, the 353-byte MTU-constrained recursion bomb (PoC) exhibits fundamentally different structural characteristics. To achieve deep ASN.1 structural nesting within a highly constrained byte footprint, the adversary is forced to repetitively inject identical structural markers, such as the \texttt{0x02} choice index. When the 64-byte sliding window enters this malicious padding region, the localized repetition causes the $SQ$ to spike to exactly 4096 ($64^2$), representing its theoretical mathematical maximum. This profound $>14\times$ signal gap (4096 versus 286) allows the SSRF to establish a highly resilient detection threshold ($\theta$). By exploiting this structural necessity of the attack, our mechanism effectively isolates algorithmic complexity threats without triggering false positives on legitimate traffic, successfully decoupling defense accuracy from expensive semantic parsing operations.

\subsection{QoS Impact of Parser Exhaustion}

We first evaluated the baseline QoS performance of the test bench without any adversarial traffic. In the nominal state, the Vanetza parser achieves a mean processing latency of 0.0151 ms with a P99 tail latency of 0.0311 ms, representing our QoS reference. Under steady-state adversarial traffic, the Unpatched Native parser (Mode 0) demonstrates severe performance degradation. As shown in Table \ref{tab:master}, at a 10\% attack intensity, the P99 latency reaches 0.1787 ms (a six-fold increase over baseline) while the Max latency of 1.7783 ms indicates significant Head-of-Line (HoL) blocking. Under CWE-674 recursion, the recursive object expansion forces the simulation pipeline to wait for the parser thread to complete the exhaustive allocation, stalling the emulated pipeline. The integration of the proposed SSRF (Mode 0 + Filter) effectively isolates the parser from these structural bombs, reducing the P99 tail latency to 0.0370 ms and the P99.9 latency to 0.1330 ms, closely approximating the baseline distribution. This confirms the efficacy of streaming structural scanning in maintaining tail latency control under high-load adversarial conditions.

\subsection{Defense-in-Depth Comparison}

Table \ref{tab:master} details the performance of Mode 1 (Official Patch). While the application-layer patch prevents infinite recursion, it introduces a highly predictable inspection cost. The error-handling logic and recursion depth checks contribute to a slightly higher median latency (0.0148 ms vs. 0.0139 ms baseline). However, the patch successfully prevents extreme outliers, though it still exhibits a high Max latency of 4.3267 ms under 10\% attack intensity due to the fixed cost of rejecting deep structures. The proposed hybrid defense outperforms both by combining the edge-layer circuit breaker with the application-layer patch (Mode 2), achieving the lowest overall Mean latency (0.0162 ms) and clamping the Max latency to 1.0796 ms. The $\mathcal{O}(1)$ SSRF demonstrates a lower amortized overhead compared to the full application-layer patch by identifying anomalies before the bit-stream is fully expanded, reducing the total CPU cycles consumed per malicious packet.

\subsection{First-Miss Penalty and Hybrid Defense}

A critical finding in our system jitter analysis is the "First-Miss Penalty." This phenomenon occurs when a structural attack packet is injected during the probabilistic sampling window but is not selected for inspection. This packet reaches the parser, triggering a high-latency event before the PRB-FSM can deplete its budget and transition to a higher risk state. In the \textit{Unpatched + Filter} configuration, this resulted in a Max latency spike of 0.8032 ms (Table \ref{tab:master}).

The \textit{Patched + Filter (Hybrid)} configuration (Mode 2) represents the most resilient architecture. By combining the edge-layer circuit breaker with the application-layer patch, we achieve the lowest overall Mean latency (0.0150 ms). The circuit breaker handles over 90\% of the adversarial load with low overhead, while the official patch serves as a fallback to mitigate the impact of any packets that bypass the probabilistic sampling window. This defense-in-depth approach helps maintain that the emulated workload remains stable even under stateful adversarial traffic designed to probe sampling gaps.

\section{Discussion}

The experimental results validate the efficacy of the proposed Parser-Layer Circuit Breaker in maintaining latency stability under structural complexity attacks. However, a robust systems-security evaluation requires a nuanced discussion of the mechanism's boundaries, inherent trade-offs, and practical deployment considerations within future MEC environments.

\paragraph{Jitter and Inspection Cost} Traditional defenses require line-rate DPI, which imposes a constant processing overhead on every packet, increasing jitter for safety-critical V2X services. By using an FSM with probabilistic sampling, we amortize this baseline inspection cost. During nominal operations ($S_0$ state), the effective complexity is reduced from $\mathcal{O}(N)$ to $\mathcal{O}(R_{sample} \times w)$, where $w$ is the constant window size, minimizing the background resource footprint. This low-overhead filtering approach is critical for high-throughput edge nodes where compute resources are heavily contested.

\paragraph{Tail Latency Control} In V2X, the P99 tail latency is often more critical than the mean. Structural attacks specifically target the "tail," causing extreme outliers that can disrupt cooperative awareness. Our results show that the circuit breaker mechanism effectively clamps the tail latency, preventing HoL blocking from cascading through the parser processing pipeline. By discarding OER structural bombs before recursive expansion using the bounded-cost SSRF, the system significantly improves tail-latency containment for legitimate time-sensitive packets.

\paragraph{False Positives and Operational Tolerance} The constant-memory SSRF relies on the statistical improbability of long sequences of identical bytes in high-variation payloads like signed OER certificates. While highly effective for ASN.1 OER, this heuristic may exhibit false positives if applied to protocols that natively utilize extensive structural padding. In vehicular systems, false positives carry inherent safety risks, as dropping legitimate DENM or CAM messages degrades cooperative awareness. Consequently, the repetition threshold ($\theta$) must be carefully tuned to the empirical distribution of the target protocol to heavily favor a fail-open operational stance for ambiguous traffic. Future deployments must balance the QoS-security tradeoff by integrating adaptive thresholds that prioritize emergency traffic, facilitating that the circuit breaker mitigates algorithmic DoS without inadvertently suppressing critical life-safety alerts. Furthermore, this mechanism is strictly a parser-layer defense; it does not provide universal protection against semantic logic flaws, cryptographic vulnerabilities, or broad volumetric DDoS attacks. Our empirical evaluation across all testing configurations (Table \ref{tab:fpr_fnr}) confirms that the SSRF achieves a 0.00\% False Positive Rate (FPR) under standard legitimate traffic, indicating that it does not falsely flag compliant C-ITS payloads. Under active structural attacks, the False Negative Rate (FNR) is maintained below 0.07\% under a 10\% attack intensity (rising to at most 1.48\% under a sparse 1\% attack due to the transition budget latency before FSM escalation). This extremely low leakage rate ensures that the parser remains protected against sustained exploitation.

\paragraph{First-Miss Penalty \& Hybrid Strategy} A limitation of probabilistic sampling is the "first-miss penalty," where an initial attack packet passes the sampling window before FSM transition, causing a transient latency spike. We advocate a hybrid strategy: deploying our edge-layer circuit breaker to filter the majority of the adversarial load, while keeping application-layer static recursion limits as a fallback to catch any bypassed packets and suppress latency spikes.

\paragraph{Stateful Adversary Behavior} Advanced attackers might employ low-frequency patterns to trigger the circuit breaker and force continuous SSRF inspection. However, the $\mathcal{O}(1)$ filter is computationally inexpensive and non-allocating. Under sustained stateful attacks, the system experiences only a minor baseline shift without HoL blocking. This confirms our capacity for \textbf{adversarial workload containment}, decoupling defense cost from attack complexity.

\paragraph{Threats to Validity} Internal validity is tied to the empirical tuning of the sampling rate and recovery budget. External validity is bounded by the use of Vanetza and IEEE 1609.2 certificates. While the SSRF logic is generalized for OER recursive patterns, evaluation under alternative encoding rules is left for future work.

\section{Limitations and Future Work}

While our circuit breaker mitigates structural attacks, our parser-centric evaluation is structured to isolate presentation-layer dynamics and does not evaluate multi-node networking contexts (e.g., ns-3).

\textbf{Adaptive Thresholding and Camouflage:} The $\mathcal{O}(1)$ SSRF relies on a statically defined repetition threshold ($\theta$). While effective for OER, which typically exhibits high bit-stream variation in legitimate cryptographic payloads, applying this heuristic to other protocols or encoding rules may require dynamic threshold tuning to prevent false positives. Furthermore, an advanced adversary could dynamically randomize structural markers---such as interleaving different ASN.1 Choice indices or injecting high-entropy padding bytes---to artificially suppress the collision concentration statistic ($SQ$) below the detection threshold $\theta$. Addressing such entropy camouflage without re-introducing high baseline processing overhead remains an open challenge. Future work will investigate adaptive thresholding mechanisms based on real-time traffic analysis, alongside the integration of lightweight, cross-window state tracking or hardware-accelerated machine learning models to detect these evasive structural anomalies reliably.

\textbf{Hardware Acceleration:} While our software-level ONNX \cite{ONNX} % TODO: Verify Bai 2019 ONNX open neural network exchange repository/specification
integration successfully bypasses Python runtime bottlenecks, the initial $\mathcal{O}(1)$ scan could be further optimized. For practical MEC deployment, future work entails migrating the structural pattern detection directly to hardware-accelerated environments, such as FPGA smartNICs or eBPF/XDP frameworks, to provide true line-rate filtering.

\textbf{Predictive Risk Modeling:} The probabilistic nature of the PRB-FSM inherently introduces a ``first-miss penalty'' under sophisticated, low-frequency pulse attacks designed to exploit the $S_0$ sampling gap. However, our architecture approaches this not as a critical flaw, but as a deliberate engineering trade-off prioritizing overall system resilience. By combining the edge-layer circuit breaker with application-layer static constraints, the system successfully downgrades what would be catastrophic, continuous Head-of-Line (HoL) blocking into isolated, transient jitter events. This ensures robust fault containment. Future research will explore predictive risk modeling to further minimize this transient jitter while strictly maintaining ultra-low latency availability in V2X environments.

\section{Conclusion}

The integration of MEC into V2X networks introduces critical vulnerabilities at the presentation layer, where algorithmic complexity attacks can exhaust parser resources and induce severe Head-of-Line (HoL) blocking. This paper presented a Parser-Layer Circuit Breaker designed to protect V2X infrastructure from ASN.1 OER structural complexity attacks. By combining an $\mathcal{O}(1)$ bounded-cost SSRF with a Probabilistic Risk Budget FSM (PRB-FSM), we demonstrated a defense mechanism that preserves QoS through \textbf{adversarial workload containment}.

Our results confirm that by shifting the defensive objective to \textbf{bounded computational exposure}, the system can substantially reduce the impact of payload algorithmic complexity on presentation-layer processing. The proposed Hybrid Defense strategy reduces P99 tail latency from 0.1793 ms to 0.0320 ms under high-intensity structural attacks, improving tail-latency containment for safety-critical V2X services. This systems-security approach provides a practical, low-overhead framework for securing high-throughput vehicular processing pipelines against parser-layer resource exhaustion.
