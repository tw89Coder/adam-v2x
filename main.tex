\begin{abstract}
Edge computing infrastructure faces severe service availability degradation when incoming data streams introduce asymmetric processing costs, where localized computational spikes exhaust finite CPU resources. Existing defenses often rely on stateful inline inspection, which mitigates these threats but violates the strict microsecond latency budgets of resource-constrained edge nodes. Instead of prioritizing inspection accuracy at the cost of performance, we propose a computation-aware edge admission architecture designed to preserve deterministic service availability, thereby mitigating queue buildup and tail-latency inflation under bursty workloads. By formulating edge admission as a bounded computational-cost problem, we establish a fundamental entropy--depth tradeoff under physical MTU constraints. This enables a zero-allocation, $\mathcal{O}(1)$ data-plane admission mechanism that leverages the $F_2$ frequency moment for inline structural filtering. To prevent inspection fatigue during nominal workloads, an asynchronous control plane utilizes a Finite State Machine (FSM) and Deep Reinforcement Learning (DRL) to optimize sampling rates dynamically. Using ASN.1 OER recursive parsing as a motivating case study, we demonstrate that our framework maintains bounded fast-path overhead and reduces P99 tail latency from 0.2229 ms to 0.0035 ms under severe workloads, ensuring resilient Quality of Service (QoS) for real-time edge applications.
\end{abstract}

\section{Introduction}
The edge computing paradigm is defined by strict resource constraints and microsecond-level service availability requirements. However, when processing heterogeneous, multi-protocol data streams, edge nodes frequently encounter payloads with asymmetric processing costs—such as deeply nested serialization structures—where processing the payload consumes disproportionately more CPU cycles than transmitting it. These localized computational spikes induce severe Head-of-Line (HoL) blocking, degrading the Quality of Service (QoS) for all subsequent time-sensitive edge applications. Protecting edge availability requires neutralizing these high-variance workloads before they exhaust finite processor resources.

Traditional admission control regulates volumetric network traffic (e.g., bits per second) but fails to address this computational heterogeneity. Existing cybersecurity defenses rely on stateful inline inspection (e.g., Deep Packet Inspection) directly in the data plane, which introduces processing latency that inherently violates edge performance budgets. To preserve deterministic service availability, the system must perform computation-aware admission control, guaranteeing $\mathcal{O}(1)$ execution overhead prior to stateful processing.

To resolve this, we formulate edge admission as a computation-aware admission problem under asymmetric processing costs. By defining a fundamental entropy--depth tradeoff, we establish a zero-allocation structural admission threshold capable of rejecting high-variance workloads at the edge ingress. Based on this formulation, we realize a decoupled admission architecture separating the latency-critical data plane (fast-path filtering) from an asynchronous control plane (slow-path adaptive resource allocation). Using recursive serialization structures (e.g., CWE-674) as a motivating case study, we instantiate this computation-aware admission mechanism on a constrained edge testbed.

The primary contributions of this paper are:
\begin{enumerate}
    \item Formulating edge admission as a computation-aware admission problem under asymmetric processing costs, and deriving an entropy--depth bound that enables deterministic $\mathcal{O}(1)$ structural admission while bounding service-time variance;
    \item Proposing a decoupled architecture that separates latency-critical structural filtering in the data plane from asynchronous resource optimization in the control plane;
    \item Demonstrating empirically that this admission mechanism mitigates localized computational spikes, preserving deterministic service availability and reducing P99 tail latency.
\end{enumerate}

\section{Problem Formulation}

\subsection{CWE-674 Multi-Protocol Bottleneck}
Recursive structures are endemic to widely used serialization schemas \cite{Ref_OER_Edge_2024}. When an edge node receives a payload with anomalous nesting depth, the presentation-layer parser attempts to allocate objects and traverse the schema tree \cite{Wang2024}. In an algorithmic complexity attack, an adversary injects syntactically valid but deeply nested structures constrained only by the Maximum Transmission Unit (MTU). This exploits the structural markers of the protocol (e.g., choice indices or container lengths) to exhaust CPU cycles prior to semantic validation.

\subsection{Queueing Theoretic Impact of Algorithmic Complexity}
The systemic impact of these computational spikes on edge service availability is best modeled using an M/G/1 queue. According to the Pollaczek-Khinchine (P-K) formula, the expected queueing delay $W_q$ is given by:
\begin{equation}
W_q = \frac{\lambda E[S^2]}{2(1-\rho)}
\end{equation}
where $\lambda$ is the arrival rate, $\rho$ is the average CPU utilization, and $E[S^2]$ is the second moment (variance) of the service time $S$. An algorithmic-complexity payload acts as a computational bomb that exponentially inflates $E[S^2]$. Equation 1 mathematically demonstrates that even if the aggregate CPU utilization $\rho$ remains strictly bounded, this massive variance alone guarantees severe Head-of-Line (HoL) blocking, causing unbounded tail latency for all subsequent legitimate edge tasks. Neutralizing this variance in $\mathcal{O}(1)$ time prior to parsing is therefore fundamental to preserving service availability.

\subsection{Limitations of Traditional Defenses}
Conventional mitigation strategies rely on Deep Packet Inspection (DPI) or static compiler patching \cite{Zhang_Edge_2018}. However, DPI involves heavy signature matching that inflates the baseline inspection cost ($C_{inspection}$). Moreover, managing bespoke inline DPI rules for diverse edge communication protocols is computationally prohibitive for a unified edge ingress layer \cite{SmartNIC_2019, Sadiq_DataPlane_2023}. 

Our security objective is to design a protocol-agnostic pre-parsing filter that operates in $\mathcal{O}(1)$ time relative to packet length, decoupling the defense from specific protocol state machines. We base this on the insight that malicious structural repetition sharply contrasts with the higher entropy of legitimate data.

While Machine Learning (ML) enables adaptive resource allocation \cite{SACD_2025}, inline ML classification introduces inference latency that fundamentally violates sub-10 ms V2X constraints. To preserve deterministic edge availability, our architecture restricts complex ML inference strictly to the asynchronous slow path. The $\mathcal{O}(1)$ data plane evaluates structural risk instantly, while the control plane updates admission thresholds at coarse-grained intervals, decoupling intelligence from the latency-critical ingress pipeline.

\section{Theoretical Foundation (Entropy--Depth Tradeoff)}

The theoretical foundation of our defense relies on the principle of structural marker concentration. Recursive payloads necessarily pack repeated protocol-specific structural markers into the MTU budget. If an attacker attempts to evade statistical detection by injecting high-entropy bytes between these markers, they inevitably consume the available physical packet capacity, thereby limiting the maximum achievable recursion depth. 

To formalize this physical boundary, we must establish the transport-layer conditions of vehicular networks. Crucially, IEEE 802.11 standards explicitly disallow MAC-layer fragmentation for broadcast and multicast frames. Since V2X safety messages (e.g., CAM and DENM) are transmitted exclusively as unacknowledged broadcast frames under the ITS-G5 protocol suite, payloads cannot be fragmented across multiple physical frames \cite{IEEE80211_2020}. This structural transport guarantee, rather than a mere simplifying approximation, allows us to define the following system assumptions:
\begin{itemize}
    \item \textbf{Assumption 1 (Finite Packet Size / MTU Limit):} The communication link has a strictly bounded Maximum Transmission Unit ($M$), and all parsed payloads must fit within a single physical frame.
    \item \textbf{Assumption 2 (Online Processing):} The edge node processes incoming packet streams on-the-fly, disallowing stateful, multi-pass deep packet inspection due to microsecond-level latency constraints.
    \item \textbf{Assumption 3 (Legacy-Compatibility Recursive Parsing):} The vulnerability exists within a retained legacy-compatibility parsing path in the decoder. While the current protocol schema is flattened and depth-bounded, backward compatibility with pre-flattened certificate chain grammars requires unbounded recursive-descent TLV traversal. The attack surface persists strictly due to this legacy code path, not the active deployed schema.
\end{itemize}

\textbf{Theorem 1 (Entropy--Depth Tradeoff under MTU Constraints).}
\textit{
Let $M$ denote the effective MTU of the underlying link and let $S_{\mathrm{syn}}$ be the minimum structural bytes required for one recursive encoding layer. To ensure that no sliding inspection window of size $w$ simultaneously contains two adjacent recursive markers, the spacing between consecutive markers must satisfy $L_{\mathrm{spacing}}\ge w$.
}

\textit{
When random cryptographic bytes or entropy padding are used as the spacing mechanism to dilute structural concentration, the required padding satisfies $S_{\mathrm{entropy}} \ge L_{\mathrm{spacing}} \ge w$.
}

\textit{
Therefore, the maximum achievable recursion depth is physically bounded by:
}
\begin{equation}
D_{\max} \le \frac{M}{S_{\mathrm{syn}}+S_{\mathrm{entropy}}} \le \frac{M}{S_{\mathrm{syn}}+w}.
\label{eq:entropy_depth_bound}
\end{equation}

\textbf{Proof.}
Let $\mathbb{I}$ be the indicator of structural detection. Evasion ($\mathbb{I}=0$) strictly requires adjacent recursive markers to be separated by at least $w$ bytes to prevent simultaneous capture within the sliding window, hence $L_{\mathrm{spacing}} \ge w$. To satisfy this spacing constraint using non-structural padding, the payload must inject $S_{\mathrm{entropy}} \ge L_{\mathrm{spacing}} \ge w$ bytes per recursive step. Consequently, the minimum byte length per recursive layer is $\min(L_{layer}) = S_{\mathrm{syn}} + w$. For a payload of recursion depth $D$, the total allocated size is bounded below by $D \cdot \min(L_{layer})$. By Assumption 1, the aggregate payload is strictly bounded by the single-frame MTU limit $M$. Thus, $D(S_{\mathrm{syn}} + w) \le M$, mathematically establishing the upper bound in Eq.~(\ref{eq:entropy_depth_bound}).
\hfill$\blacksquare$

Theorem 1 establishes that recursive amplification inevitably manifests as localized structural concentration under finite message budgets. Therefore, detecting localized collision concentration becomes sufficient for identifying malicious recursive payloads. Among available streaming statistics, the second frequency moment ($F_2$) provides an $\mathcal{O}(1)$ estimator of concentration without logarithmic computation, making it suitable for edge deployment.

\caption{Pareto frontier between entropy padding and the maximum achievable recursive depth under MTU constraints. Increasing entropy padding to evade detection necessarily reduces the achievable recursion depth, revealing a fundamental entropy--depth tradeoff.}

This entropy--depth tradeoff is the cornerstone of our defense. It guarantees that any structurally concentrated recursive payload will be detected, while any payload padded with sufficient entropy to evade detection will be too shallow to cause CPU exhaustion.

\subsection{O(1) Streaming Structural Risk Filter}
To identify anomalous structural concentration, we utilize the second frequency moment ($F_2$) as an engineering primitive \cite{Alon1996}. The $F_2$ moment is a classic streaming algorithm metric that estimates collision concentration in bounded memory. We compute the localized $F_2$ score, denoted as $SQ$, over a sliding window of size $w$:
$$SQ = \sum_{j \in \mathcal{V}} c_j^2$$
where $c_j$ is the frequency of byte value $j$ in the window. 

For legitimate high-entropy payloads, $c_j$ is highly dispersed, resulting in a low $SQ$. Conversely, recursion bombs inflate the $F_2$ moment due to concentrated structural markers. 

To prevent false positives, the $SQ$ calculation explicitly masks the \texttt{0x00} state. In specific schema-driven serializations like ASN.1 OER, unpopulated optional fields generate dense \texttt{0x00} sequences. This pruning eliminates protocol-specific benign noise without degrading the theoretical bound, because forcing an uncontrolled recursion fundamentally requires active (non-zero) structural markers to traverse the schema tree.

\begin{algorithm}[!ht]
\caption{Bounded Streaming Structural Risk Filter}
\label{alg:bssrf}
\begin{algorithmic}[1]
\Require Payload $P$, window size $w$, scanning limit $L_{max}$, threshold $\theta$
\State Initialize frequency sketch $H[256] \gets 0$
\State Initialize fixed-size ring buffer $W[w]$ and index $idx \gets 0$
\State $SQ \gets 0$ \Comment{Localized $F_2$ moment}
\State $n_{scan} \gets \min(|P|, L_{max})$

\For{$i = 0$ to $n_{scan} - 1$}
    \State $b \gets P[i]$
    \If{$i \ge w$}
        \State $b_{old} \gets W[idx]$
        \State $SQ \gets SQ - (2H[b_{old}] - 1)$
        \State $H[b_{old}] \gets H[b_{old}] - 1$
    \EndIf
    \State $W[idx] \gets b$
    \State $H[b] \gets H[b] + 1$
    \State $SQ \gets SQ + (2H[b] - 1)$ \Comment{Amortized $F_2$ update}
    \State $idx \gets (idx + 1) \pmod{w}$

    \If{$i \ge w$ and $SQ > \theta$}
        \State \Return \textbf{True} \Comment{Structural anomaly detected}
    \EndIf
\EndFor
\State \Return \textbf{False}
\end{algorithmic}
\end{algorithm}

Algorithm \ref{alg:bssrf} updates $SQ$ in $\mathcal{O}(1)$ time per byte. By limiting the inspection to a parametric maximum length $L_{max}$ and utilizing a threshold $\theta$, the filter overhead is strictly $\mathcal{O}(1)$ relative to the packet length, requiring constant memory and zero dynamic allocation. This C++ data plane implementation ensures hardware-level optimization suitable for latency-critical paths.

While the $\mathcal{O}(1)$ filter is highly efficient, continuously inspecting all traffic can lead to inspection fatigue on resource-constrained nodes. We introduce a Probabilistic Risk Budget Finite State Machine (PRB-FSM) to dynamically modulate the inspection sampling rate based on a continuous risk budget $B_t$ \cite{Sosnowski2024}. Budget thresholds (normalized design boundaries, e.g., $\tau_1=70$, $\tau_2=40$) map the continuous budget to discrete operational states ($S_0 \dots S_3$) with escalating sampling rates. Importantly, because these thresholds represent a normalized reference scale, their specific numeric values are arbitrary scaling choices; the DRL agent in the cognitive plane automatically calibrates its learned policy parameters (e.g., the base sampling rate) to adapt to these boundary states, rendering the system's dynamic control invariant to absolute threshold scales.

To automatically tune this sampling budget under non-stationary traffic, we formulate the traffic adaptation as a Constrained Markov Decision Process (CMDP) \cite{Altman1999_CMDP}. Let the telemetry state at control interval $t$ be $\mathbf{s}_t = [p_t, \bar{q}_t, \rho_t]^{\mathsf T}$, where $p_t$ is the current nominal sampling rate, $\bar{q}_t$ is the normalized structural variance, and $\rho_t$ is the observed anomaly ratio. The controller selects a discrete rate adjustment $a_t \in \{-0.10, -0.05, 0, 0.05, 0.10\}$ to dynamically update the PRB-FSM boundaries. To preserve availability under constrained CPU budgets, the system must minimize data-plane inspection overhead while guaranteeing that the exploit leakage rate $\eta_{leak}(t)$ remains below a strict security threshold $\eta_{target}$:
\begin{equation}
\min_{\pi} \mathbb{E}_{\pi} \left[ \sum_{t} \gamma^t p_t \right] \quad \text{s.t.} \quad \mathbb{E}_{\pi} \left[ \eta_{leak}(t) \right] \le \eta_{target}
\end{equation}

By applying a Lagrangian relaxation, we decouple the continuous optimization problem from its specific solver:
\begin{equation}
L(\pi, \lambda) = \mathbb{E}_{\pi} \left[ \sum_{t} \gamma^t \left( -p_t - \lambda [\eta_{leak}(t) - \eta_{target}]^+ \right) \right]
\end{equation}
where $\lambda > 0$ is the adaptive penalty multiplier. Crucially, this formulation restricts complex mathematical optimization to the asynchronous slow path. To solve this continuous problem in our current testbed, we deploy a Deep Q-Network (DQN) as a practical solver to approximate the optimal policy. The specific deep learning architecture acts only as an interchangeable policy optimizer and is strictly decoupled from the latency-critical data path. If the cognitive plane suffers from exploration noise, the data plane maintains a hardcoded Layer-1 Overrun Guard that immediately overrides the policy to 100\% sampling upon precipitous budget depletion \cite{Nygard2007}, ensuring deterministic lower bounds on edge availability.

\section{Experimental Evaluation}
The entropy--depth analysis motivates the use of a bounded structural concentration metric in the data plane. The following experiments evaluate whether this theoretically motivated design achieves the intended systems objective—preserving availability under constrained CPU budgets.

\subsection{Experimental Testbed and Traffic Modalities}
The testbed utilizes a 12th Gen Intel i7-12700H and 16 GB of RAM running Ubuntu 22.04 LTS. Single-core pinning is adopted to isolate algorithmic CPU overhead from multi-core scheduling effects, establishing a controlled baseline for resource-constrained edge environments. The software is based on an unpatched version of Vanetza\footnote{Vanetza C-ITS Suite (evaluated at unpatched commit \texttt{[3a18193]}): \url{https://github.com/riebl/vanetza}}, an open-source ETSI C-ITS suite processing CAM/DENM payloads via the \texttt{asn1c} compiler\footnote{asn1c (Mouse Fork): \url{https://github.com/mouse07410/asn1c}}.~To evaluate the defense, exploit payloads were constructed via structured fuzzing targeting the legacy-compatibility certificate-chain parsing path in Vanetza. It is important to note that this vulnerable path coexists with a newer depth-bounded parser, and no CVE has been assigned to this specific finding to date. We integrated a custom QoS evaluation framework into the ingress pipeline to measure presentation-layer latency. Crucially, the $\mathcal{O}(1)$ algorithmic complexity and zero-allocation memory design of the filter provide instruction-set-independent advantages. To further validate this architectural portability across heterogeneous edge environments, hardware-specific evaluation on resource-constrained ARM-based AIoT platforms (e.g., Raspberry Pi) is actively underway.

To establish a realistic baseline, we extract nominal background traffic from the V2AIX dataset \cite{10920150}. We interleave this real-world traffic with 1,000 mutated recursive malicious payloads across multi-scale injection rates ranging from 0.1\% to 10.0\%. The system is evaluated under four scheduling configurations: Mode 0 (Continuous Uniform), Mode 1 (Single Strong Pulse), Mode 2 (Periodic Pulsing). We measure the Mean, Median, P99, and Max Latency to quantify queue delay and Head-of-Line (HoL) blocking.

\subsection{Edge Workload Amplification and Filter Efficacy}
To quantify the vulnerability, we profiled the unpatched parser under MTU-constrained structural attacks. In our isolated micro-benchmarks, we observed that parsing latency exhibits strict linear growth ($\mathcal{O}(N)$) relative to recursion depth, bounded only by the physical MTU. To profile the worst-case scenario, a single MTU-saturating exploit payload (approx. 1400 bytes) was measured to consume approximately 960 $\mu$s of CPU time. As predicted by the Pollaczek-Khinchine equation (Eq. 1), this millisecond-scale anomaly exponentially inflates the service time variance $E[S^2]$. In real-time AIoT networks, such localized computational spikes instantly induce Head-of-Line (HoL) blocking, degrading the QoS of subsequent safety messages.

Under these workloads, our deployed pre-filter successfully isolates malicious structural concentration from benign structural padding. By rejecting payloads that violate the theoretical entropy--depth bound, the filter maintains a 0.00\% False Positive Rate (FPR) across all clean test sets while enforcing $\mathcal{O}(1)$ execution boundaries, thereby preempting the inflation of $E[S^2]$.

Under the Lagrangian reward formulation, the agent achieves stable convergence within 1,200 training episodes, establishing a robust control policy without runtime instability. To evaluate runtime attack mitigation under this optimized policy, we trace the latency timelines under Mode 1 (single pulse) and Mode 2 (periodic pulsing) attacks.

This stability is enforced by the FSM's hysteresis-based recovery, which mandates a clean streak of 1,000 packets before restoring the lower sampling policy, preventing oscillation exploits and bounding the transient "first-miss penalty" spikes.

\section{Conclusion}
We propose an ultra-lightweight, $\mathcal{O}(1)$ streaming detector for recursive parser amplification on resource-constrained edge systems. By formalizing the entropy--depth tradeoff under physical MTU limits, we demonstrate that structural concentration is fundamentally unavoidable for recursive exploits. Our architecture decouples a fast-path $F_2$ data plane filter from a slow-path DRL sampling optimizer. This prevents CPU exhaustion during severe floods (reducing P99 latency from 0.2229 ms to 0.0035 ms) while minimizing inspection fatigue during sparse threats. As this structural vulnerability extends to industrial protocols like IEC 61850 GOOSE \cite{IEC61850_Goose_L2, Fuzzing_IEC61850}, our adaptive defense mechanism offers broad applicability for preserving real-time QoS in critical infrastructure.
